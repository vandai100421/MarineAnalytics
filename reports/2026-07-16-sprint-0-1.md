## 1. Tổng quan

Hai sprint đầu tiên tập trung vào scaffolding toàn bộ project và xây dựng pipeline ingestion AIS hoàn chỉnh. Kết thúc Sprint 1, hệ thống đã có khả năng:

- Kết nối WebSocket tới AISStream.io, nhận message AIS realtime
- Decode đầy đủ các loại message AIS (type 1/2/3/5/18/19/24)
- Lưu vị trí vào TimescaleDB hypertable + cache Redis
- Upsert thông tin tĩnh tàu (vessels) với ship_type_name
- Tự động reconnect khi WS đứt (exponential backoff)
- Thu thập metrics (msg/sec, decode error, positions written)
- CI pipeline chạy ruff + mypy + pytest + eslint + tsc

**Chi tiết decode (T1.1):**
- **Type 1/2/3** (PositionReport): lat, lon, sog, cog, heading, nav_status, rot
- **Type 18** (PositionReportInt): Class B CS position, không có nav_status
- **Type 19** (StandardClassBPositionReport): position + static data (name, ship_type, callsign, dimensions) → viết cả position và vessel
- **Type 5** (ShipStaticData): name, callsign, ship_type, imo, dimensions, destination, ETA
- **Type 24** (StaticDataReport): name, callsign, ship_type, dimensions, destination
- Thêm `ship_type_name` decode từ AIS ship type code (Fishing, Cargo, Tanker, Passenger...)
- Thêm fallback MMSI từ `UserID` field
- Thêm ETA parsing

**Chi tiết metrics (T1.7):**
- `IngestionMetrics` class: messages_received, decode_errors, positions_written, vessels_upserted, messages_by_type
- Periodic log mỗi 60 giây
- Snapshot msg/interval để tính msg/sec
