**SSE Architecture:**
- `SubscriberManager`: quản lý set subscriber, mỗi subscriber có queue (maxsize 500), bbox filter, min_sog filter
- Broadcaster loop: mỗi `SSE_BATCH_INTERVAL_SECONDS` (1s) đọc tất cả Redis positions → filter theo subscriber bbox/sog → push JSON batch vào queue
- SSE endpoint: `EventSource` stream với events `connected`, `positions`, `ping`, `error`
- Backpressure: queue full → drop + log warning; max 200 clients; inactive subscriber auto-remove
- Heartbeat: `sse-starlette` ping + custom `TimeoutError` → `ping` event mỗi 15s

**Continuous Aggregate:**
```sql
CREATE MATERIALIZED VIEW vessel_counts_hourly
WITH (timescaledb.continuous) AS
SELECT time_bucket('1 hour', ts) AS bucket, mmsi,
       count(*) AS report_count, avg(sog), max(sog)
FROM position_reports GROUP BY bucket, mmsi
WITH NO DATA;
```
- Auto-refresh policy: mỗi 1 giờ, refresh window 2h → 5min

**Stats API:**
- `GET /api/v1/stats/overview`: active_vessels (Redis count), total_vessels (DB count), avg_sog
- `GET /api/v1/stats/by-type`: group by ship_type_name, order by count desc
- `GET /api/v1/stats/heatmap`: sample 10000 lat/lon points (Phase 2 will enhance)
